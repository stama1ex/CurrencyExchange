# Исправление синхронизации баланса касс

## Проблема
Когда пользователь добавляет, изменяет или удаляет следующие операции:
- **Инкассации** (пополнение/снятие денег)
- **Операции обмена валют** (обмен валют на кассе)

Баланс касс и отчеты не обновляются. Таблицы операций обновляются локально, но данные в отчетах и рассчитанный баланс не синхронизируются.

## Решение

### 1. **BalanceSyncService** (`service/BalanceSyncService.java`)
Новый сервис для расчета баланса касс. Основные методы:
- `calculateBalance(cashDeskId, currencyCode)` - рассчитывает баланс для конкретной валюты
  - Учитывает только **завершенные** операции (статус = "Выполнена")
  - "Пополнение" добавляет средства (+)
  - "Инкассация" вычитает средства (-)
- `calculateAllBalances(cashDeskId)` - рассчитывает баланс по всех валютам

**Важно:** Баланс рассчитывается только для операций со статусом "Выполнена".

### 2. **MainController** (обновлено)
Теперь выполняет роль **центрального уведомителя** об обновлении данных:
- `setReportController(controller)` - регистрирует ReportController
- `notifyReportUpdate()` - статический метод, вызываемый другими контроллерами для уведомления об изменениях

### 3. **ReportController** (обновлено)
- Регистрирует себя в MainController при инициализации
- Добавлен метод `refreshReport()` - публичный метод для обновления отчета из других контроллеров
- Логика загрузки отчета перемещена в `loadReportInternal()` для переиспользования

### 4. **IncassationController** (обновлено)
Все CRUD методы теперь вызывают синхронизацию:
- `addIncassation()` → `MainController.notifyReportUpdate()`
- `updateIncassation()` → `MainController.notifyReportUpdate()`
- `deleteIncassation()` → `MainController.notifyReportUpdate()`

### 5. **ExchangeOperationController** (обновлено)
Все CRUD методы теперь вызывают синхронизацию:
- `addOperation()` → `MainController.notifyReportUpdate()`
- `updateOperation()` → `MainController.notifyReportUpdate()`
- `deleteOperation()` → `MainController.notifyReportUpdate()`

## Схема работы

```
Пользователь добавляет инкассацию
         ↓
IncassationController.addIncassation()
         ↓
Запись в БД
         ↓
refreshTable() (локальная таблица обновляется)
         ↓
MainController.notifyReportUpdate() ← НОВОЕ
         ↓
ReportController.refreshReport()
         ↓
loadReportInternal() 
         ↓
Отчет обновляется в БД view
(cash_desk_status, incassations_view и т.д.)
```

## Что обновляется

При изменении операций:
1. **Локальные таблицы** (в текущей вкладке) обновляются сразу
2. **Отчет "Статус касс" (cash_desk_status)** обновляется автоматически
3. **Отчет "Инкассации" (incassations_view)** обновляется автоматически
4. **Отчет "Операции обмена" (exchange_operations_view)** обновляется автоматически

Если пользователь находится на вкладке "Отчеты", отчет обновится в реальном времени.
Если пользователь находится на других вкладках, отчет обновится при переходе на вкладку "Отчеты".

## Примечания

1. **Условие синхронизации:** Баланс учитывает только операции со статусом "Выполнена"
2. **Типы операций:** 
   - "Пополнение" (TOP_UP) - добавляет баланс
   - "Инкассация" (INCASSATION) - вычитает баланс
3. **Null-safety:** Если ReportController еще не инициализирован, уведомление игнорируется (не вызывает ошибку)
4. **Производительность:** `refreshReport()` обновляет только текущий отчет, не все отчеты сразу

## Тестирование

Рекомендуемые тесты:
1. Добавить инкассацию (пополнение) со статусом "Выполнена" → проверить, обновился ли отчет "Статус касс"
2. Добавить инкассацию (снятие) со статусом "Выполнена" → проверить баланс
3. Добавить операцию обмена → проверить отчет "Операции обмена"
4. Изменить статус операции на "Отменена" → баланс не должен учитывать отмененную операцию
5. Удалить операцию → откат баланса к предыдущему состоянию
